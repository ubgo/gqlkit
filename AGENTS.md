# Repository Guidelines for AI Agents & Developers

**CRITICAL CONTEXT:** this project is a **code generator**. Most `.go` / `.ts` files under any `sdk/` directory are generated output, not source.

1. **NEVER** hand-edit generated code. That means anything under `example-*/sdk/`, the generated `mockapi/graph/generated.go` + `mockapi/graph/model/models_gen.go`, or any file whose header says `Code generated by gqlkit. DO NOT EDIT.`
2. **ALWAYS** change the **source** and regenerate:
   - Generator behavior → edit the templates in `gqlkit/pkg/templater/template/*.tmpl` and the generators in `gqlkit/pkg/clientgen/` (Go) / `gqlkit/pkg/clientgents/` (TypeScript), then regenerate every example.
   - Example SDKs → edit the schema, then `go run ./cmd/generate` in that example (or the matching `task` target).
   - The mockapi schema → edit `mockapi/graph/*.graphqls`, then `task mockapi:generate`.

## Key configuration files

* **`go.work`** — defines the modules in the workspace. Tests run each module in isolation with `GOWORK=off`.
* **`Taskfile.yml`** — the registry of every runnable command (`task --list`). Check it before proposing shell scripts.
* **`config.jsonc`** (per example) — custom scalar-to-language-type bindings for the generator.

## Documentation map

* **`README.md`, `docs/`, per-module `README.md`** — human-facing guides (setup, workflows, API usage). Public, GitHub-rendered plain markdown.
* **`CLAUDE.md`** — the imperative project rules for AI agents (commit etiquette, invariants, workflow). Read it first.
* **`.claude/AI_CONTEXT.md`** — a dense technical context dump (type-mapping rules, package graph, design decisions) optimized for LLM token efficiency.
* **`CONTRIBUTING.md`** — architecture diagram, module map, and the dev/test/release workflow.

## Git commit rules

* **NEVER** include "Generated with Claude Code" or similar AI attribution in commit messages.
* **NEVER** include "Co-Authored-By: Claude" or any AI co-author lines.
* Commit subjects are sentence case, ≤70 chars, with **no** Conventional-Commit prefix (`feat:` / `fix:`). Explain the *why* in the body; keep messages focused on the change.
